# خروجی‌های مالی و کسورات — قرارداد API

ماژول واحد مالی: ساخت **فایل کسورات ماهانه** با تجمیع خودکار اقساط وام و حق‌بیمهٔ تکمیلی، سپس نهایی‌سازی و خروجی CSV.
همه‌چیز با دسترسی `finance.export.view` و RLS بر اساس محدودهٔ سازمانی. پایهٔ آدرس‌ها `/api/v1` بدون اسلش انتها.

## فایل‌های کسورات

### `POST /finance/batches` — ساخت فایل جدید
```json
{ "period": "1405-03", "title": "کسورات خرداد", "org_unit": 5 }
```
`org_unit` اختیاری (خالی = کل کشور). وضعیت اولیه `draft`.

### `GET /finance/batches` · `…/{id}` — فهرست/جزئیات (صفحه‌بندی، RLS)
واحد مالی استان فقط فایل‌های محدودهٔ خودش را می‌بیند.

### `POST /finance/batches/{id}/generate` — تولید خودکار اقلام
از **وام‌های پرداخت‌شده** (`disbursed`) قسط ماهانه، و از **بیمه‌نامه‌های تأییدشده** حق‌بیمه را برای پرسنلِ محدودهٔ فایل استخراج و درج می‌کند. `total_amount` و `item_count` بازمحاسبه می‌شوند. فقط روی فایل `draft`.

### `POST /finance/batches/{id}/add-item` — افزودن قلم دستی
```json
{ "personnel": 12, "amount": 500000, "description": "جریمه" }
```

### `GET /finance/batches/{id}/items` — اقلام فایل (صفحه‌بندی)

### `POST /finance/batches/{id}/finalize` — نهایی‌سازی (قفل فایل) → `finalized`
فایل بدون قلم قابل نهایی‌سازی نیست (`400`).

### `GET /finance/batches/{id}/export` — خروجی CSV (فایل قابل دانلود)
پاسخ یک فایل `text/csv` با BOM (سازگار با اکسل فارسی) شامل ستون‌های شماره‌پرسنلی/نام/منشأ/مرجع/مبلغ/شرح و ردیف جمع کل.
اگر فایل `finalized` باشد، پس از خروجی به `exported` تغییر می‌کند.

## چرخهٔ وضعیت
`draft` → (generate/add-item) → (finalize) `finalized` → (export) `exported`

## داشبورد
`finance.monthly_deductions` = جمع کل آخرین فایل کسورات در محدودهٔ کاربر (RLS).
فهرست `finance.exports` از `GET /finance/batches` تغذیه می‌شود.

## نکته
حق‌بیمهٔ درج‌شده فعلاً برابر `premium_total` بیمه‌نامه است (ساده‌سازی: یک قسط ماهانه).
در صورت نیاز به تقسیط سالانه، منطق `generate_items` قابل تنظیم است.
