# تست‌ها (pytest)

## نصب و اجرا
```bash
source .venv/bin/activate
pip install -r requirements-dev.txt
pytest                 # کل تست‌ها
pytest -v              # با جزئیات
pytest tests/test_hr.py            # یک فایل
pytest -k rls          # فقط تست‌های مرتبط با RLS
```

## نکته‌های پیکربندی
- `pytest.ini` با `--no-migrations` اجرا می‌شود: اسکیمای دیتابیس تست مستقیماً از مدل‌ها ساخته می‌شود،
  پس برای تست **نیازی به وجود فایل‌های migration نیست** و همیشه با مدل‌های فعلی هماهنگ است.
- کدِ OTP در تست‌ها با یک fixture (`fixed_otp`) ثابت روی `123456` می‌شود تا جریان ورود قابل‌تست باشد.
- بیشتر تست‌ها با `force_authenticate` کاربر را احراز می‌کنند؛ تست‌های `test_auth_otp.py`
  خودِ جریان واقعی OTP→JWT را از طریق endpointها می‌زنند.

## پوشش فعلی (۴۱ تست)
- **auth/OTP:** درخواست، عدم نشت کد در پاسخ، موبایل نامعتبر، محدودیت نرخ، تأیید موفق/ناموفق،
  کاربر ناشناس، ثبت لاگ ورود، الزام احراز هویت روی endpoint محافظت‌شده.
- **RBAC:** `me/roles` و flatten دسترسی‌ها، گیتِ super-admin روی `permissions`، ساخت نقش، حفظ scope.
- **مدیریت کاربر (super_admin):** فهرست صفحه‌بندی، منع کاربر عادی، ساخت کاربر + انتساب/حذف نقش،
  موبایل نامعتبر/تکراری، دسترسی به لاگ ورود.
- **داشبورد:** فیلتر منو بر اساس نقش (employee vs super_admin)، بخش‌های داشبورد،
  مقدار واقعی `hr.total_personnel`، و `pending` ماندن آمارِ ماژول‌های نساخته.
- **hr:** فهرست استان، اعتبارسنجی کد ملی (رد/قبول)، **RLS** (کاربر استانی فقط پرسنل خودش)،
  دیدِ کامل super_admin، رد کد ملی تکراری در افراد تحت تکفل.
- **اقامت:** ساخت مجموعه، منع بدون دسترسی، چرخهٔ تغییر وضعیت + صف نظافت + mark-cleaned،
  پنل پلان، و **RLS** روی مجموعه‌ها.
- **seed:** صحت تعداد دسترسی/نقش/داشبورد/ویجت/استان/امکانات، idempotent بودن، و کامل بودن دسترسی‌های super_admin.
